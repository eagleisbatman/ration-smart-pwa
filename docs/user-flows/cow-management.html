06 — Cow Management
Routes: /cows, /cows/new, /cows/:id, /cows/:id/edit

Cow Management is the central CRUD module for managing cattle profiles. Users can list, search, filter, add, view, edit, and delete cows. All operations support offline mode with IndexedDB caching and background sync.

Cow List
  • Fetches cows from API on mount, also loads from local IndexedDB cache
  • Search filters by name, breed, tag number, or coat color
  • Cards are clickable — navigate to /cows/:id
  • FAB (+) navigates to /cows/new
  • Empty state shows illustration + "Add your first cow" prompt
  • Selection mode button in toolbar toggles checkboxes for bulk actions
  • Bulk actions: Archive selected, Delete selected (with confirmation)

Form Fields
  • Photo: Optional. Camera or gallery upload.
  • Name: required, minimum 1 character
  • Tag Number: Optional identifier (ear tag, RFID, etc.)
  • Breed: fetched from GET /api/v1/breeds, dropdown list
  • Coat Color: Optional. 7 color options: black, white, brown, red, spotted, mixed, other.
  • Weight: required, must be > 0, in kilograms
  • Milk Yield: liters per day, optional (0 for dry cows)
  • Milk Fat %: typically 3.0–6.0%, optional
  • Lactation Stage: early (0–100 days), mid (100–200 days), late (200+ days), dry (not producing)
  • Pregnancy: toggle + month field (1–9 months)
  • Body Condition Score: Dropdown with values 1–5 (1=Emaciated, 2=Thin, 3=Moderate, 4=Heavy, 5=Obese). Helps assess cow's nutritional status.
  • Age: in months, optional
  • Activity Level: low, normal (default), high
  • Same form component used for both add and edit (route determines mode)
Offline Support
  • When offline, cow is saved to local IndexedDB
  • Queued for sync when connection restores
  • SyncStatusChip shows pending count
  • On sync: POST /api/v1/cows (new) or PUT /api/v1/cows/:id (edit)

Cow Detail
  • Overflow menu: Edit, Delete, Share
  • "Get Diet" navigates to /diet/new?cow_id={id} (pre-fills wizard step 1)
  • "Log Milk" navigates to /logs/new?cow_id={id}
  • Delete requires confirmation dialog
  • Milk history shows visual chart of daily yields
  • Edit button is a floating action button (FAB) in bottom-right
  • Yield trend shows percentage change vs last week
  • HealthEventTimeline component shows chronological health events
  • Recent Milk Logs table shows last few days of morning/evening milk data
Data Relationships
  • Each cow belongs to a farmer profile (farmer_id)
  • Cow can have many milk logs
  • Cow can have many diet plans
  • Breed comes from master breed list
  • When editing, form pre-fills all fields from cow data
