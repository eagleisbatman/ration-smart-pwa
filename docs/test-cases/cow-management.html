TC-COW-01 View Cow List High

Preconditions

User is logged in and at least 1 cow exists in the system. The app is open and the user has access to the /cows page.

Steps

  1. Navigate to the Cows tab from the bottom navigation bar or sidebar menu.
  2. Observe the cow list page. Verify that cow cards are displayed in a scrollable list.
  3. Inspect each cow card and confirm it shows the cow's name, breed, milk yield (liters/day), and lactation stage (e.g., Early, Mid, Late, Dry).
  4. Verify that search and filter options are visible at the top of the list page.

Expected Result

The cow list page displays all registered cows as individual cards. Each card shows the cow's name, breed, current milk yield, and lactation stage. Search and filter controls are visible and accessible at the top of the page.

Acceptance Criteria

  • Cow cards display name, breed, milk yield, and lactation stage
  • Search input field is visible at the top of the list
  • Filter options (breed, lactation stage) are visible and accessible
  • List is scrollable when more cows exist than fit on screen
TC-COW-02 Add New Cow with All Required Fields Critical

Preconditions

User is logged in and on the /cows page. The device is online so the breed dropdown can fetch data from the API.

Steps

  1. Tap the (+) floating action button to open the "Add New Cow" form.
  2. Optionally tap the Photo upload area at the top to add a cow photo. The app offers a choice between Camera (take a new photo) and Gallery (choose an existing image). This field is optional.
  3. Enter a name for the cow in the Name text field (required, e.g., Lakshmi).
  4. Optionally enter a Tag Number (ear tag, RFID, or other identifier, e.g., IN-MH-0042).
  5. Tap the Breed dropdown and select a breed from the list populated by the backend API (e.g., Holstein Friesian).
  6. Optionally select a Coat Color from the dropdown. Seven options are available: Black, White, Brown, Red, Spotted, Mixed, Other.
  7. Enter the cow's weight in the Weight (kg) numeric input field (required, must be greater than 0 and no more than 1500, e.g., 450).
  8. Optionally enter the current milk production in the Milk Yield (L/day) field (e.g., 12).
  9. Optionally enter the milk fat percentage in the Milk Fat % field (valid range 0–10%, e.g., 3.5).
  10. Select a Lactation Stage from the dropdown: Early, Mid, Late, or Dry.
  11. If the cow is pregnant, toggle the Pregnancy switch to ON and enter the Pregnancy Month (1–9). The month field appears only when the toggle is ON.
  12. Select a Body Condition Score (BCS) from the dropdown. Options: 1 – Emaciated, 2 – Thin, 3 – Moderate, 4 – Heavy, 5 – Obese.
  13. Optionally enter the cow's Age in months (e.g., 36).
  14. Select an Activity Level from the dropdown: Low, Normal, or High.
  15. Optionally enter any additional information in the Notes textarea.
  16. Tap the Save button to create the cow record.

Expected Result

The cow is created successfully in the system. The app redirects back to the cow list page at /cows. The newly created cow appears in the list with the correct name, breed, milk yield, and lactation stage displayed on its card. All optional fields that were filled in are persisted and visible on the cow detail page.

Acceptance Criteria

  • The "Add New Cow" form opens when the (+) button is tapped
  • Photo upload is optional and offers Camera or Gallery as source
  • Name is required; form cannot be submitted without it
  • Tag Number is optional and accepts free text
  • Breed dropdown is populated with breeds from the backend API
  • Coat Color dropdown offers 7 options: Black, White, Brown, Red, Spotted, Mixed, Other
  • Weight is required, must be >0 and ≤1500 kg
  • Milk Yield (L/day) is optional
  • Milk Fat % is optional, valid range 0–10%
  • Lactation Stage dropdown provides Early, Mid, Late, Dry
  • Pregnancy toggle reveals month field (1–9) only when ON
  • BCS is a dropdown with values 1–5 (1=Emaciated, 2=Thin, 3=Moderate, 4=Heavy, 5=Obese)
  • Age in months is optional
  • Activity Level dropdown provides Low, Normal, High
  • Notes is an optional textarea
  • After saving, the app redirects to the cow list at /cows
  • The new cow appears in the list with correct data on its card
  • No error messages are displayed during or after creation

TC-COW-06 Body Condition Score Dropdown (1–5) Medium

Preconditions

The user is on the Add New Cow or Edit Cow form. The BCS dropdown component is visible in the form.

Steps

  1. Locate the Body Condition Score (BCS) dropdown on the add/edit cow form. BCS measures the cow's fat and muscle condition on a scale of 1 (emaciated) to 5 (obese). Most healthy dairy cows score between 2.5 and 3.5.
  2. Tap the BCS dropdown to open the list of options.
  3. Verify the dropdown displays five options: 1 – Emaciated, 2 – Thin, 3 – Moderate, 4 – Heavy, 5 – Obese.
  4. Select BCS from dropdown (e.g., 3 – Moderate) and verify the selected value is displayed in the dropdown field.
  5. Change the selection to another value (e.g., 4 – Heavy) and verify the dropdown updates accordingly.

Expected Result

The BCS dropdown shows options: 1 – Emaciated, 2 – Thin, 3 – Moderate, 4 – Heavy, 5 – Obese. Selecting a value updates the dropdown display. The selected BCS value is saved correctly with the cow record. Glossary note: "BCS measures the cow's fat/muscle condition on a scale of 1 (emaciated) to 5 (obese). Most healthy dairy cows score 2.5–3.5."

Acceptance Criteria

  • BCS is presented as a dropdown (not a slider)
  • Dropdown shows 5 options with values 1 through 5
  • Each option includes a descriptive label: 1=Emaciated, 2=Thin, 3=Moderate, 4=Heavy, 5=Obese
  • Selecting an option updates the dropdown display to show the chosen value
  • The selected BCS value is saved correctly with the cow record
TC-COW-07 Lactation Stage Selection Medium

Preconditions

The user is on the Add New Cow or Edit Cow form. The lactation stage selector is visible in the form.

Steps

  1. Locate the Lactation Stage selector on the add/edit cow form. Lactation stage describes how long since the cow last gave birth (calved) and started producing milk.
  2. Tap the Early option and verify it is selected. Early lactation is 0–100 days after calving, when milk production is typically at its peak.
  3. Tap the Mid option and verify it becomes selected (and Early is deselected). Mid lactation is 100–200 days after calving.
  4. Tap the Late option and verify it becomes selected. Late lactation is 200+ days after calving, when milk production naturally declines.
  5. Tap the Dry option and verify it becomes selected. A dry cow is not currently producing milk, typically in the final 2 months before the next calving.

Expected Result

Four lactation stage options are available: Early (0–100 days after calving), Mid (100–200 days), Late (200+ days), and Dry (not currently producing milk). Only one option can be selected at a time. Glossary: "Lactation stage = how long since the cow last gave birth and started producing milk."

Acceptance Criteria

  • Exactly 4 lactation stage options are available: Early, Mid, Late, Dry
  • Only one option can be selected at a time (single-select behavior)
  • Selecting a new option automatically deselects the previous one
  • The selected lactation stage is saved correctly with the cow record
TC-COW-08 Pregnancy Toggle Reveals Month Field Medium

Expected Result

When the pregnancy toggle is switched ON, a pregnancy month input field appears, accepting values from 1 to 9. When the toggle is switched OFF, the month input field is hidden from view. The form dynamically shows or hides the month field based on the toggle state.

Acceptance Criteria

  • Pregnancy toggle defaults to OFF with no month field visible
  • Toggling ON reveals the pregnancy month input field
  • Pregnancy month field accepts values from 1 to 9
  • Toggling OFF hides the pregnancy month field
  • The pregnancy status and month are saved correctly with the cow record

TC-COW-10 Create Cow Offline — Syncs When Online High

Preconditions

The device is offline (airplane mode enabled or WiFi disconnected). The user is logged in and on the /cows page. The app has been previously used while online so that cached data (e.g., breeds) is available.

Steps

  1. Confirm the device is offline (no internet connection). Check the browser or device status indicator.
  2. Tap the (+) button to open the "Add New Cow" form.
  3. Fill in the required fields: name (e.g., Offline Cow), breed (from cached list), weight (e.g., 400), milk yield (e.g., 10), fat% (e.g., 4.0), and lactation stage (e.g., Mid).
  4. Tap Save. Verify the cow is saved locally and appears in the cow list, potentially with an offline indicator (e.g., a cloud icon with a line through it, or a "pending sync" badge).
  5. Reconnect the device to the internet (disable airplane mode or reconnect WiFi).
  6. Wait for the app to automatically sync the locally saved cow to the backend. Observe any sync indicator or notification.
  7. After sync completes, verify the cow now appears without the offline indicator, confirming it has been saved to the backend server.

Expected Result

The cow is saved locally when the device is offline, and an offline indicator is shown on the cow card. When the device reconnects to the internet, the app automatically syncs the cow to the backend. After syncing, the offline indicator is removed and the cow exists in both the local and remote databases.

Acceptance Criteria

  • Cow can be created and saved while the device is offline
  • The cow appears in the local cow list with an offline/pending sync indicator
  • When the device goes back online, the cow auto-syncs to the backend
  • After sync, the offline indicator is removed from the cow card
  • The cow record exists on the backend server after sync (verifiable via API or admin panel)
