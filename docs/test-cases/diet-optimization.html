TC-DIET-01 Complete 5-Step Wizard — Happy Path Critical

Preconditions

User is logged in. At least 1 cow exists in the system. At least 3 feeds exist. Device is online. Diet optimization = calculating the best mix of feeds to meet a cow's nutritional needs while minimizing cost or maximizing milk production.

Steps

  1. Navigate to the Diet tab and tap New to begin the diet optimization wizard.
  2. Step 1 — Select Cow: Choose an existing cow from the dropdown list.
  3. Step 2 — Animal Details: Verify the following fields are auto-filled from the cow record: weight, milk yield, fat %, lactation stage, pregnancy status, BCS (Body Condition Score — dropdown with values 1–5: 1=Emaciated, 2=Thin, 3=Moderate, 4=Heavy, 5=Obese), and age.
  4. Step 3 — Feed Selection: Use the search bar to find feeds. Select at least 3 feeds from the list.
  5. Step 4 — Optimization Goal: Choose "Minimize Cost" as the optimization goal.
  6. Step 5 — Review: Verify the review page displays all inputs (cow, animal details, feeds, goal). Tap "Generate Diet".

Expected Result

The diet is generated successfully. The results page displays: an ingredient table with amounts in kg/day, a daily cost total, and a nutritional summary showing crude protein %, TDN % (Total Digestible Nutrients), calcium, and phosphorus values.

Acceptance Criteria

  • All 5 wizard steps are completed without errors
  • Results page shows ingredient amounts in kg/day for each selected feed
  • Total daily cost is displayed in the local currency (INR)
  • Nutritional summary includes crude protein %, TDN %, calcium, and phosphorus
  • No error banners or loading failures occur during generation
TC-DIET-02 Manual Cow Entry Mode High

Preconditions

User is on the diet wizard Step 1 (Select Cow). The wizard has just been opened via the New button.

Steps

  1. On Step 1, toggle to "Enter Manually" instead of selecting an existing cow from the dropdown.
  2. Proceed to Step 2 (Animal Details).
  3. Verify that all animal detail fields are empty and editable: weight, milk yield, fat %, lactation stage, pregnancy status, BCS (dropdown, values 1–5), and age.
  4. Fill in all fields manually with valid values.

Expected Result

All animal detail fields in Step 2 are editable and initially empty. No cow is pre-selected. The user can fill in all required values manually and proceed to Step 3.

Acceptance Criteria

  • Toggle to "Enter Manually" mode is available on Step 1
  • No cow is pre-selected when manual mode is active
  • All 7 animal detail fields are empty and editable in Step 2
  • User can proceed to Step 3 after filling all fields

TC-DIET-11 Generate Diet — Results Page Critical

Preconditions

All 5 wizard steps are complete with valid inputs. Device is online. User is on Step 5 (Review) with the "Generate Diet" button enabled.

Steps

  1. Tap the "Generate Diet" button on the review step.
  2. Observe the processing status banner with a spinner and "Processing diet…" text while the API processes the request.
  3. If generation fails, verify a red banner appears with an error message and a "Try Again" button.
  4. If generation is pending (queued), verify a waiting banner is displayed.
  5. Once completed, verify the results page displays a feed breakdown table with columns: feed name, quantity (kg/day), and cost per day.
  6. Verify the total daily cost is displayed below the feed breakdown table.
  7. Verify nutrient balance progress bars are shown for: CP (Crude Protein), TDN (Total Digestible Nutrients), and DM Intake (Dry Matter Intake), each displaying actual vs required values.
  8. Verify progress bar color coding: green = meets requirement, orange = slightly below, red = deficient.
  9. Verify a Recommendations section is present with suggestions from the optimization engine.
  10. Verify a Warnings section is displayed if any nutrients are deficient.
  11. Verify a "Regenerate" button is available on the results page.

Expected Result

Status banners reflect the current diet state: processing (spinner with "Processing diet…"), failed (red banner with error message and "Try Again" button), or pending (waiting banner).

On success, the results page displays: a feed breakdown table (feed name, quantity in kg/day, cost/day per feed), a total daily cost, nutrient balance progress bars for CP, TDN, and DM Intake showing actual vs required values with color coding (green = meets requirement, orange = slightly below, red = deficient), a Recommendations section from the optimization engine, and a Warnings section if any nutrients are deficient. A "Regenerate" button is available for re-running the optimization.

Acceptance Criteria

  • Processing banner with spinner is visible during API call
  • Failed status shows red banner with error message and "Try Again" button
  • Pending status shows waiting banner
  • Feed breakdown table shows feed name, quantity (kg/day), and cost per day for each feed
  • Total daily cost is displayed prominently
  • Nutrient balance progress bars are shown for CP, TDN, and DM Intake with actual vs required values
  • Progress bars use color coding: green (meets requirement), orange (slightly below), red (deficient)
  • Recommendations section is present with optimization engine suggestions
  • Warnings section appears when nutrients are deficient
  • "Regenerate" button is present and tappable on the results page
TC-DIET-12 Regenerate Diet from Previous High

Preconditions

At least 1 diet result already exists in the system. The user is viewing the results page of a previously generated diet.

Steps

  1. On the diet results page, tap the "Regenerate" button.
  2. Verify the diet wizard opens with all previous inputs pre-filled: the cow (or manual entry data), animal details, selected feeds, and optimization goal.
  3. Check the browser URL for a regenerateFrom query parameter referencing the previous diet.
  4. Modify one or more inputs (e.g., change a feed selection or adjust a weight value).
  5. Proceed through the wizard and tap "Generate Diet" again.

Expected Result

The diet wizard opens with all previous inputs pre-filled from the existing diet. The URL contains a regenerateFrom query parameter. The user can modify any input and regenerate the diet. A new diet result is created without overwriting the original.

Acceptance Criteria

  • Wizard opens with cow, animal details, feeds, and goal pre-filled from the previous diet
  • URL contains regenerateFrom query parameter
  • All pre-filled values are editable
  • Regenerated diet creates a new result (does not overwrite the original)
